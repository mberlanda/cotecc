point for the design iteration (responsive redesign, smarter AI, new game
motions). Work was delivered as five PRs:

1. `baseline/api-hardening` — cotecc-api hardening
2. `baseline/app-deps` — CoteccApp dependency cleanup
3. `baseline/ai-player` — deterministic point-avoiding AI
4. `baseline/web-screenshots` — browser target + screenshot collection (stacked on 2)
5. `baseline/remove-web-skeleton` — this PR

## 3. De-duplication / dependency diet

- Removed unused deps from CoteccApp: `@react-native-community/masked-view`
  (deprecated upstream), `react-native-reanimated` (never imported and its
  babel plugin was never configured), `expo-status-bar` (never imported).
- Removed `body-parser` from cotecc-api (duplicate of built-in
  `express.json()`).
- Removed the whole `cotecc-web` package (CRA skeleton with zero app
  code); the browser target is CoteccApp itself via `react-native-web`.
- Screenshot tooling (Playwright) lives in `tools/screenshots` with its own
  `package.json` so it never enters an app bundle.

## 4. AI player

`aiMoveToPlay` is now fully deterministic (no `Math.random`) and plays the
point-avoidance game:

- ducks with the highest card that still loses the led suit;
- when forced to win and **last to act**, dumps its strongest card
  (position passed via the new `playersInRound` argument);
- counts cards seen in past turns: leads the weakest card opponents can
  still beat and never leads guaranteed winners;
- when void, discards the most dangerous card (points, then rank,
  tie-break towards emptying short suits).

Next steps for "smarter design": difficulty levels (the rule layer makes
this easy — drop rules to weaken it), capot awareness, and inferring
opponents' voids from off-suit plays.
